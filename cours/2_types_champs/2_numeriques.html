Les numériques

Les entiers

De l'anglais integer, ils sont représentés par le mot clef INT. Il existe aussi des déclinaisons qui permettront de déterminer la taille de l'entier (les valeurs qu'il peut prendre) et surtout sa place en mémoire comme dans le tableau suivant :

Type Nombre d'octets Minimum Maximum
TINYINT 1 -128 127
SMALLINT 2 -32 768 32 767
MEDIUMINT 3 -8 388 608 8 388 607
INT 4 -2 147 483 648 2 147 483 647
BIGINT 8 -9 223 372 036 854 775 808 9 223 372 036 854 775 807

Autres particularités de ce type, nous pouvons lui préciser différentes choses.

  • S'il est signé ! Autrement dit, aura-t-il des valeur négatives ? Si ce n'est pas le cas on précisera UNSIGNED et notre entier ira de 0 à 255 pour un TINYINT.
  • On peut aussi préciser le nombre de chiffre minimum à l'affichage. Par exemple je veux afficher au moins 2 chiffres pour un INT, je saisirai INT(2).
  • Enfin avec le pécédent point se combine généralement avec l'attribut ZEROFILL, qui ajoutera des zéros à gauche de notre INT(2), pour qu'il respecte la taille d'affichage définie, donc 01 pour avoir saisie 1 dans notre cas.

Les décimaux

A l'instar des entier, nous avons plusieur façons de définir des décimaux, et deux sous-catégories : les flottants et les fixes.

Les flottants

Le type FLOAT est utilisé pour représenter des données numériques approchées. MySQL permet de définir la taille et le nomde de décimales, mais ce n'est pas dans la norme SQL. Par exemple si je déclare un FLOAT(7,4), j'aurai un FLOAT de 7 digits dont 4 représenterait la décimale : 999.9999.

De la même manière on peut déclarer deux autres types, le DOUBLE (pour double précision) et le REAL. Le DOUBLE et par définition plus précis que le REAL ; mais MySQL a décidé de stocker les deux sur 8 octets, donc ce n'est pas totalement vrai. Par soucis de compatibilité, utilisez le DOUBLE sans spécifier de paramètres.

Les fixes

Les deux "fixes" existants sont considérés identiques en MySQL et sont utilisées par des valeurs dont il est primordial de conserver la précision exacte, comme pour des données financières. On les déclare en précisant leurs paramètres : DECIMAL(5,2) signifie 5 digits dont 2 après la virgule, 1654,32 par exemple.

ATTENTION : on constate bien des différences entre le SQL et MySQL ! Pensez à vous référer à la documentation !
