Le Typage

Pourquoi le typage ?

Une base de données (BDD) peut être comparée à un fichier Excel. Elle comporte des tables comme un fichier Excel contient des feuilles, qui sont composés de colonnes aussi appelés champs.
Ces champs pourront stocker différents types d'informations et, comme dans Excel on déterminerait le format (type) de la cellule, on peut déterminer le type de champs en fonction des informations que l'on souhaite stocker. Plus que pouvoir, on DOIT déterminer le type des champs d'une table de base de données.

Le typage est important pour plusieurs raisons :

  • Eviter un gaspillage de mémoire (ex. : si vous stockez de toutes petites données dans une colonne faite pour stocker de grosses quantités de données)
  • Limiter les problèmes de performance (ex. : il est plus rapide de faire une recherche sur un nombre que sur une chaîne de caractères)
  • Empécher un comportement contraire à celui attendu (ex. : trier sur un nombre stocké comme tel, ou sur un nombre stocké comme une chaîne de caractères ne donnera pas le même résultat)
  • Permettre d'utiliser des fonctionnalités propres à un type de données (ex. : stocker une date comme une chaîne de caractères vous prive des nombreuses fonctions temporelles disponibles)

Il est donc important de bien choisir le meilleur type possible !
